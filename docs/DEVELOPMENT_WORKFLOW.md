# Development Workflow

## 1. Branching Strategy
- **Main Branch**: `main`. Protected. Production-ready code only.
- **Feature Branches**: `feat/description-of-work`.
- **Fix Branches**: `fix/issue-description`.
- **Chore Branches**: `chore/deps-update`.

## 2. Pull Request (PR) Rules
- **Granularity**: Small PRs (<400 lines) preferred.
- **Verification**:
  - `pnpm lint` must pass.
  - `pnpm type-check` must pass.
  - `pnpm build` must pass.
- **Description**: Must link to the Milestone or Task ID being addressed.

## 3. Human Review
- Code generated by AI Agents (Antigravity/Claude) **MUST** be reviewed by a human before merging to `main`.
- **Focus**:
  - Does this introduce new npm dependencies? (Avoid bloat).
  - Does this modify `ui` package unnecessarily?
  - Does this follow the Offline/Sync patterns?

## 4. Handling Requirement Changes
- If a requirement changes (e.g., "Add Payments"), **STOP**.
- Update `/docs/PRODUCT_REQUIREMENTS.md` first.
- Update `/docs/DATA_MODEL.md` second.
- Only then write code.
- **Rule**: Documentation is the specification. Code is the implementation. Code without spec is tech debt.

## 5. Known Compatibility Pins
- **Dependency Pins**:
  - `bcrypt==4.0.1`: Pinned due to `passlib` compatibility issues (see [Sprint 1 Integration Gate](./INTEGRATION_GATE_SPRINT1.md)).
- **Infrastructure**:
  - **PostgreSQL Port**: Runs on host port `5433` because default port `5432` is in use. Check `docker-compose.yml`.

## 6. Troubleshooting
- **Next.js Cache Issues**: If you see "Cannot find module" errors or 404s for `/_next/static/*`, run the clean reset script:
  ```bash
  ./scripts/reset-web-dev.sh
  ```
  This will kill hanging processes, clear caches, and restart the dev server.
